About Myself

Frontend Developer


I am a front-end developer who enjoys building websites and interfaces that feel clear and enjoyable to use. My career goal is to work in web design and create experiences that look good and make sense for real users. I am learning how to use modern tools like responsive design, accessibility standards, and front-end frameworks because the web keeps changing and I want to keep growing with it.

I am interested in how new ideas in UX, design systems, and performance shape the way people interact with digital products, and I want to build work that feels current and useful.

It also keeps me curious, which is something I try to bring into my work. It reminds me to stay open-minded and pay attention to the small details that make an experience memorable.

Explore my Work

Anime Library

Anime Library is a personal project from 2024 where I designed and coded an anime catalog website with filters and search. The goal was to practice building a clean layout that lets users quickly scan cover art, titles, and key details without feeling overwhelmed. I focused on using HTML, CSS, and JavaScript to create a simple browsing flow, including category filters and a responsive grid. This piece is about presenting a large amount of visual content in an organized way so fans can discover new shows comfortably on desktop or mobile screens. It reflects my focus on simple anime discovery.

Mandatory.GG Clone

Mandatory GG Clone is a 2024 web design piece inspired by a gaming news and esports site. I recreated the overall structure and feel using only my own HTML and CSS to practice layout, hierarchy, and typography. The project focuses on big hero images, match highlights, and featured articles that feel energetic but still readable. I paid attention to card spacing, color contrast, and hover states to support quick scanning. This piece communicates how design can deliver a bold, competitive mood while still keeping navigation and content clear for different kinds of viewers. It also shows my interest in esports.

YouTube Clone

YouTube Clone is a 2024 practice project where I rebuilt a simplified video platform interface. Using HTML and CSS, I focused on a clean layout, clear spacing, and a responsive grid that adapts from desktop to smaller screens. The design highlights thumbnails, titles, and channel details so users can quickly decide what to watch next without confusion. I treated this piece as a way to study familiar patterns in modern media sites and to show that I can recreate them with attention to detail while still keeping usability, alignment, and visual balance in mind. It also improved how I design.

Minesweeper

Minesweeper is a 2025 browser game where I recreated the classic grid puzzle using HTML, CSS, and JavaScript. My goal was to practice working with game logic, conditional checks, and a dynamic grid that responds to user clicks. Tiles reveal numbers, bombs, or empty spaces, and the interface updates in real time based on player choices. I kept the visuals minimal so the focus stays on clarity and interaction at all times. This piece communicates how code can turn simple rules into tension and satisfaction while also showing careful handling of states and edge cases. This project strengthened my debugging.

Scalable Tic-Tac-Toe

Scalable Tic Tac Toe is a 2025 project where I expanded the standard three by three game into a grid that can grow larger. Built with HTML, CSS, and JavaScript, it includes basic styling plus custom logic for placing moves and checking winners on different board sizes. I experimented with random behavior and layout spacing to keep play from feeling predictable. The project communicates my interest in taking a simple idea and pushing it further through code while still keeping the visuals readable and the interactions easy to understand for curious players. It gave me practice structuring reusable game logic.

Pokémon Battle Clone is a 2025 turn based battle demo that I built with HTML, CSS, and JavaScript.
